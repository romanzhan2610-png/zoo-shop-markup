# 🐾 Интернет-магазин зоотоваров «Дружок» — Frontend-часть

Адаптивная, семантическая верстка многостраничного интернет-магазина с продуманной логикой UI/UX. Проект разработан на Vanilla JS с модульной архитектурой и полностью подготовлен для интеграции с любой CMS или бэкенд-фреймворком.

## 🛠 Технологический стек

- **Сборка:** Vite + vite-plugin-handlebars (компонентный подход, использование partials для шапки, футера и модальных окон).
- **Стилизация:** CSS3 (Variables, Flexbox/Grid, BEM-подобная методология). Полностью кастомная стилизация элементов форм (радио-кнопки, чекбоксы, range-слайдеры).
- **Логика:** Vanilla JavaScript (ES6+, модульная архитектура).
- **Интеграции:**
  - `Swiper.js` — адаптивные и кроссбраузерные слайдеры.
  - `IMask.js` — маскирование полей ввода (телефоны).
  - `Yandex Maps API` — кастомизированная интерактивная карта с филиалами.
- **Оптимизация:** Использование глобальных SVG-спрайтов (`partials/sprite.html`) для мгновенной загрузки иконок.

## 👨‍💻 Инструкция для Backend-разработчика

Фронтенд спроектирован с учетом минимизации боли при «натяжке» на сервер.

### 1. Mock API (Заглушки запросов)
Вся симуляция работы с сервером (авторизация, регистрация, восстановление пароля, проверка промокодов) вынесена в единый файл `js/api/mockApi.js`. 
Вам не нужно искать асинхронную логику по всему коду — достаточно заменить `setTimeout` заглушки в этом файле на реальные `fetch` или `axios` запросы к вашему бэкенду. UI (лоадеры, вывод ошибок) отреагирует автоматически.

### 2. Тестовые данные для UI
Для проверки успешных сценариев в текущей верстке зашиты следующие данные:
- **Авторизация:** Логин: `test@mail.ru` | Пароль: `123456`
- **Регистрация / Восстановление:** Правильный код из СМС: `1111`
- **Корзина (Промокоды):** `TEST` или `SALE`

### 3. Data-атрибуты для интеграции
В верстке заложены `data-атрибуты` для привязки логики корзины и карточек товаров. При рендеринге шаблонов на сервере подставляйте реальные ID:
- Кнопка «В корзину»: `<button class="btn-add-cart-desktop" data-product-id="123">`
- Выбор веса/объема: `<span class="product-var" data-variant-id="123-500g">500 г</span>`
- Элемент корзины (для удаления): `<div class="cart-item" data-product-id="123">`

## 🚀 Запуск и сборка проекта

1. Установите зависимости:
   `npm install`

2. Запустите dev-сервер с Hot Module Replacement (HMR):
   `npm run dev`

3. Соберите проект для продакшена (билд появится в папке /dist):
   `npm run build`

4. Деплой на GitHub Pages (автоматически пушит папку dist):
   `npm run deploy`
   
P.S. В корне проекта лежит скрипт context.js — утилита для быстрой генерации текстового слепка кодовой базы (полезно при работе с LLM и нейросетями).
